# Superflow Changelog

Superflow-specific release notes for the AFFiNE 0.26.3 fork. Upstream
AFFiNE release notes live at https://affine.pro/blog?tag=Release%20Note —
this file documents what's different in `mygogocash/Superflow`.

## v1.10.2 — 2026-05-04

**Live Gmail/Drive importers + license unlock + AI write tools.** Image:
`asia-southeast1-docker.pkg.dev/affine-495114/affine/affine-gogocash:v1.10.2`.

### Added

- **Gmail import** (`packages/backend/server/src/plugins/google-oauth/gmail.service.ts`).
  Search your Gmail and import messages as new AFFiNE docs. Subject
  becomes the doc title; `From: <sender> · <date>` italics header; HTML
  body decoded to plaintext via a 5-step regex stripper (no
  `sanitize-html` dep). Reuses the same `DocWriter.createDoc` path the
  AI `doc-write` tool uses, so the imported doc lives in the standard
  Y-Doc/storage adapter pipeline.
- **Drive picker** (`packages/backend/server/src/plugins/google-oauth/drive.service.ts`).
  Search your Drive, copy `webViewLink` to clipboard, or open in Drive.
  No auto-import of file content (Drive's MIME zoo is out of scope).
- **OAuth token refresh** with 5-minute leeway.
  `GoogleOAuthService.getValidAccessToken(userId, workspaceId, scope)`
  refreshes against `oauth2.googleapis.com/token` whenever the stored
  token is within 5 minutes of expiry; persists rotated refresh tokens
  via `IntegrationConnectionModel.updateTokens`.
- **AI permission mode picker** in the chat panel (Read-only / Edit
  current doc / Full agent). Maps to new `AIToolsConfig` flags:
  `editingDocs`, `composingDocs`, `editingDataViews`. Backend write
  tools (`docEdit`, `sectionEdit`, `docCreate`, `docCompose`,
  `dataViewFilter`, `dataViewAutofillColumn`) gate on these flags.
  "AI made changes" chip rendered when a write tool fires.
- **Gmail + Google Drive integration cards** in Settings → Integrations.
  Pre-import scaffold (connect/disconnect/connected-as state) shipped
  in v1.10.1; live import dialogs shipped in v1.10.2 (this version).

### Changed

- **License tab hidden** on self-hosted. The "Get teams plan" upgrade
  panel and seat indicator (`Members 2/10`) are gone. Backend
  enforcement: `QuotaService.getWorkspaceQuota` returns `memberLimit:
100_000` when `env.selfhosted` is true. All downstream seat checks
  (`tryCheckSeat`, `checkSeat`, member-add resolvers) silently pass.
  Search for `// SUPERFLOW:` comments to find every changed site.
- **Members panel title** no longer shows `(N/100000)`. When
  `memberLimit >= 10000` the ratio is hidden; reads as `Members (3)`
  instead of `Members (3/100000)`.
- **`Summary as title` prompt** switched from `gpt-5-mini` to
  `gemini-2.5-flash`. Upstream default is unavailable on Superflow's
  Vertex-only stack, so chat session auto-naming was silently failing
  and every history entry stayed as "New chat" forever.
- **Auto Tag** UX: friendlier toast listing actual tag names
  (`Added 4 tags: mind, ที่ปรึกษา, innovation and 1 more` instead of
  bare counts), loading state, graceful empty/error states.

### Fixed

- **Auto Tag SSE parser garbage**. Tags were rendering as raw
  `{"type":"text-delta","textDelta":"..."}` JSON wrappers. Root cause:
  `textToText({stream: false})` in `provider/request.ts` was
  concatenating each `event.data` JSON string verbatim instead of
  parsing it and extracting `textDelta`. Fixed at two layers:
  (1) `request.ts` now JSON-parses each chunk and pulls only
  `textDelta`; (2) `tags.tsx` has a defensive 4-strategy parser plus a
  `looksLikeSseFragment` rejecter for any candidate containing
  structural fragments (`{`, `}`, `\\`, `"type"`, `"textDelta"`).
- **GraphQL `UndefinedTypeError` on `DriveFile.size`**. Backend crashed
  on startup with `Make sure you are providing an explicit type for the
"size" of the "DriveFile"`. NestJS metadata reflection cannot infer
  a GraphQL type from a union containing `null`. Fixed with explicit
  `@Field(() => String, { nullable: true })`. Same family as the
  v1.7.0 occurrence — documented in `CLAUDE.md` §5 as
  "broken TWICE now" so the next refactor catches it.

### Operational notes

- **Required env vars** for Gmail/Drive (set on the GCE VM at
  `/srv/affine/compose/.env`):
  - `GOOGLE_OAUTH_CLIENT_ID`
  - `GOOGLE_OAUTH_CLIENT_SECRET`
  - `GOOGLE_OAUTH_REDIRECT_URI` (defaults to
    `${SERVER_ORIGIN}/oauth/google/callback` if unset)
- **Required APIs** enabled on GCP project `affine-495114`:
  `gmail.googleapis.com`, `drive.googleapis.com`.
- **OAuth consent screen** must list scopes `gmail.readonly` and
  `drive.readonly`. While in Testing mode, only listed test users
  can connect; switch to Production after Google verification.
- **Build pipeline gotcha**: `Dockerfile.fullstack` copies pre-built
  `packages/*/dist/` artifacts. Always run `yarn affine bundle -p
@affine/server` and `yarn affine bundle -p web` BEFORE
  `docker buildx build`. Skipping the bundle step ships the previous
  bundle silently — no error, just no new code in the image.

### Incident

- **First v1.10.2 deploy crashed the backend on startup** — Caddy
  returned HTTP 502 to users for ~30 seconds while the rollback to
  v1.10.1 propagated. Cause: the `DriveFile.size` `@Field` annotation
  (above). Caught by the post-deploy smoke test
  (`grep -c editingDocs /app/dist/main.js` returned 0 on what should
  have been the new image — turned out the schema crash prevented
  Nest from listening). Rolled back, fixed in 1 line, re-pushed under
  the same `v1.10.2` tag, redeployed cleanly.
