# Manut Changelog

Manut-specific release notes for the AFFiNE 0.26.3 fork. Upstream
AFFiNE release notes live at https://affine.pro/blog?tag=Release%20Note —
this file documents what's different in `mygogocash/Manut`.

## v1.12.0 — 2026-05-13

**PM, CRM, and Reminders v0 frontend rollout; frontier model picker
expansion; Vertex Model Garden providers; production analytics fix.**
Image: `asia-southeast1-docker.pkg.dev/affine-495114/affine/affine-gogocash:v1.12.0`.

The previous release (v1.11.0) was the brand rename and post-rebrand
documentation cleanup. v1.12.0 is the first release on the renamed
stack and covers the feature work that landed in the same window but
was out of scope for v1.11.0's narrative.

### Added

- **Projects and Tasks workspace page** (`/projects`). Project list
  rendered as expandable cards with an active/archived badge; tasks
  appear inline per project with status, priority, due date, and a
  delete affordance. Create flows: `NewProjectModal` (name required,
  description optional) wires to `createSfProject`; `NewTaskModal`
  (title required, status/priority/dueAt) wires to `createSfTask`. An
  inline status dropdown calls `updateSfTaskStatus` and applies a
  strike-through for `DONE` / `CANCELLED`. Loading skeleton, empty
  state, and error retry are wired. Files:
  `packages/frontend/core/src/modules/manut-pm/`.
- **CRM workspace page** (`/crm`) with four tabs — Accounts, Contacts,
  Deals, Activities. Each tab is wrapped in `Suspense` plus a
  `SWRErrorBoundary` with `fallbackRender`. Accounts capture name,
  industry (free-text), website, notes. Contacts capture
  `firstName`/`lastName`, email, phone, and an account picker. Deals
  capture name, value, a stage picker with an inline "Add stage", and
  an account picker; the list is grouped and sorted by stage (no
  Kanban yet — that is a v1 follow-up). Activities capture
  `type` (`CALL` / `MEETING` / `EMAIL` / `NOTE` / `TASK`), subject,
  and body. All create flows call
  `createSfCrm{Account,Contact,DealStage,Deal,Activity}`. Files:
  `packages/frontend/core/src/modules/manut-crm/`.
- **Reminders workspace page** (`/reminders`) with three tabs — Due
  now, Upcoming, and Done & Cancelled. Tabulation reads
  `SCHEDULED|PROCESSING` plus `fireAt` against the current time for
  Due/Upcoming, and routes `COMPLETED|CANCELLED|FAILED` to Done. The
  `NewReminderModal` (title required, body optional, `fireAt`
  required) calls `createSfReminder`; "Mark done" calls
  `cancelSfReminder`. Files:
  `packages/frontend/core/src/modules/manut-reminders/`.
- **Feature-flag-gated sidebar nav** for Projects, CRM, and
  Reminders. A new `useSuperflowEnabled()` hook in
  `packages/frontend/core/src/components/root-app-sidebar/index.tsx`
  reads from `ServerService.server.features$` and renders each entry
  only when the backend has the gated module loaded. Non-Manut
  installs see nothing changed.
- **`ServerFeature.Superflow`** backend enum value. A new
  `SuperflowFeatureRegistrar` in `plugins/manut/manut.module.ts`
  calls `server.enableFeature(ServerFeature.Superflow)` on module init
  when the gate flag is true, and `disableFeature` when not. The enum
  value is left as `'superflow'` because the frontend reads it through
  the public GraphQL server-config contract; the deferred-rename plan
  in `CLAUDE.md` §9 covers this.
- **Frontier model picker expansion.** The "Chat With AFFiNE AI"
  prompt's `optionalModels` array now exposes ten models grouped by
  family — Google Gemini (2.5 Flash, 2.5 Pro, 3.1 Flash Lite Preview,
  3.1 Pro Preview), Anthropic Claude (Sonnet 4, Sonnet 4.5, Opus 4),
  and Meta Llama 4 (Scout 17B/16E, Maverick 17B/128E). Defaults are
  unchanged; users can pick the new entries from the chat model
  dropdown. The catalogue of record is `optionalModels` on
  `CHAT_PROMPT` in
  `packages/backend/server/src/plugins/copilot/prompt/prompts.ts`,
  cross-referenced in `CLAUDE.md` §5d.
- **Moonshot Kimi, xAI Grok, and Alibaba Qwen providers** wired up.
  These are the three families that aren't on Vertex Model Garden, so
  each ships as its own provider implementation (auth, config, model
  catalogue, dispatch). Files: `providers/moonshot/`, `providers/xai/`,
  `providers/alibaba/`. They are not in the chat picker by default —
  enabling them requires populating the provider config on the VM.

### Changed

- **Production domain** continued to answer on
  `https://manut.gogocash.co` after the brand rename in v1.11.0;
  v1.12.0 brings the first feature work onto the renamed stack.
- **Module gate environment variable.** `ENABLE_MANUT_MODULE=true` is
  now the preferred flag for the Manut plugin
  (`plugins/manut/manut.module.ts`). The legacy
  `ENABLE_SUPERFLOW_MODULE` value is still honored as a fallback so
  existing VM `.env` files keep working without coordination, but new
  environments should set the new name. The frontend's sidebar nav
  remains gated through `ServerFeature.Superflow` regardless of the
  env var name, because the enum value is part of the public schema.
- **Deploy script.** `scripts/vm/deploy.sh` now recreates the
  `affine_migration` job container before swapping the production
  `affine` service, so Prisma migrations always run against the new
  image. Older deploy paths sometimes left a stale migration container
  in place; the fix makes the migration step idempotent across
  redeploys.
- **Workflow paths.** `.github/workflows/superflow-*.yml` were
  renamed to `manut-*.yml` and `.docker/gogocash/` was renamed to
  `.docker/manut/`. Display names ("Manut CI", "Manut Build", etc.)
  were already correct from earlier brand work; this release brings
  the filenames into line. The `manut-release-handover.mjs` script,
  `vm/deploy.sh`, `README.md`, and `CLAUDE.md` references were updated
  in lock-step.

### Fixed

- **Analytics · Connections panel crash on production.**
  `isAnalyticsModuleEnabled()` previously read
  `process.env.DEPLOYMENT_TYPE` directly. Production compose files
  don't always set that var — the `Env` class defaults it to
  `'selfhosted'` (matching what `/info` reports), but the gate
  predicate ignored that default and returned `false` on bare envs.
  The analytics module therefore never registered, the GraphQL schema
  lacked `connections`, and the Settings → Connections panel surfaced
  "Unhandled error raised" on `manut.gogocash.co` even though the
  resolver had been on `main` since 2026-05-04. Fix: read
  `globalThis.env.selfhosted` instead. The predicate is now in a
  standalone `feature-flag.ts` so it's unit-testable without pulling
  the full module's napi-bound dependency graph. Added: a
  defence-in-depth fallback so that if the backend schema is missing
  `connections` (version skew, module disabled, etc.) the panel
  shows a one-line "ask your admin to enable the module" notice
  instead of the generic red error banner. Files:
  `packages/backend/server/src/plugins/analytics/feature-flag.ts`,
  `packages/frontend/core/src/modules/analytics/`.
- **`gpt-5-mini` poisons the auto prompt.** The auto prompt
  (injected in-memory at `PromptService` boot via `injectAutoPrompt`)
  advertised `gpt-5-mini` as a user-selectable model. Manut's Vertex
  stack has no OpenAI provider, so any user who picked GPT-mini from
  the chat dropdown got a silent failure — the `session.normalize()`
  check accepted the model because it was in `optionalModels`, routed
  to a non-existent OpenAI provider, and produced nothing. No log, no
  error toast. This is the runtime-only sibling of the same trap
  documented in `CLAUDE.md` §5c that escapes the four-step prompt
  update dance (because it's constructed via `service.ts`, not the
  static `prompts.ts` catalogue). Fix: drop `gpt-5-mini` from the
  in-memory injection and leave a comment in the array so the next
  person who wants to add OpenAI models has the context.

### Documentation

- `CLAUDE.md` §5 picked up new entries:
  - `rxjs/finnish` lint rule requires `$` suffixes on observable
    fields; trips up sub-agents who follow standard RxJS conventions.
  - Sub-agent `tsc -b` invocations produce stray `.d.ts` /
    `.d.ts.map` files inside `src/` directories — always brief
    sub-agents with `tsc --noEmit`, never `tsc -b`.
  - The chat-panel `streamObjects` array exposes a tool-call shape
    that's worth knowing about for any feature that wants to react
    to AI activity (e.g. activation buses, change badges).
- `docs/HANDOVER.md` — "Last reviewed" bumped to today; "Current
  State" and "Feature Status" sections refreshed to describe the
  shipped PM/CRM/Reminders v0 surface and the resolved analytics
  Connections crash.
- `docs/AI_SESSION_HANDOVER.md` — branch state, latest commits, and
  next-step entries refreshed for the v1.12.0 cut.
- `README.md` — feature list now mentions the new Manut workspace
  modules; production image tag bumped to the v1.12.0 line; icon
  reference uses the post-rebrand Manut mark (icons themselves were
  swapped in PR #27 and remain unchanged here).

merged at the time of this release, so this changelog does not claim
them.

### Operational notes

- **Required env vars** for the Manut module on the VM at
  `/srv/affine/compose/.env`:
  - `ENABLE_MANUT_MODULE=true` (preferred; falls back to
    `ENABLE_SUPERFLOW_MODULE=true`)
- **Required env vars** for the frontier model picker — no new vars,
  but the chat dropdown only renders entries whose backend provider
  is configured in `/srv/affine/data/affine-config/config.json`. The
  three new non-Vertex families need their own config blocks if you
  want them in the picker:
  - `copilot.providers.moonshot.{apiKey, baseUrl}`
  - `copilot.providers.xai.{apiKey, baseUrl}`
  - `copilot.providers.alibaba.{apiKey, baseUrl}`
- **Build pipeline.** `Dockerfile.fullstack` still expects pre-built
  `dist/` artifacts in the build context — `yarn affine bundle -p
@affine/server` and `yarn affine bundle -p web` must run before
  `docker buildx build`. See `CLAUDE.md` §5 "Dockerfile.fullstack
  expects PRE-BUILT dist artifacts".
- **Verification checklist** (run before tagging):
  - `yarn lint:ox` — 0 warnings, 0 errors.
  - `yarn workspace @affine/core tsc --noEmit` — 0 errors.
  - `yarn workspace @affine/server tsc --noEmit` — 0 errors.
  - `yarn workspace @affine/server ava 'src/__tests__/manut/*.spec.ts'`
    — all passing.
  - `yarn affine bundle -p web` — compiled successfully, no errors.
  - Browser smoke on `manut.gogocash.co`:
    - Sign-in mounts; `#app` has children; no `console.error`.
    - Settings → Connections renders without "Unhandled error raised".
    - `/projects`, `/crm`, `/reminders` are present in the sidebar
      when `ENABLE_MANUT_MODULE=true` and absent otherwise.
- **Rollback** is the same as any v1.11.x deploy:
  `gh workflow run manut-rollback.yml`. The Manut module ships
  additive resolvers, additive Prisma models (already migrated in
  v1.11.0's PR #26 wave), and additive frontend routes — none of
  the production-facing contracts changed in v1.12.0.

### Breaking changes

None expected. The PM/CRM/Reminders frontend is feature-flag-gated on
`ServerFeature.Superflow`; installations without the flag set are
unaffected. The frontier model additions are append-only on
`optionalModels`; the default model in the chat picker is unchanged
at `gemini-2.5-flash`. The analytics fix restores a panel that was
silently broken; no consumer of the `connections` GraphQL field
needs to change. The deploy-script fix is internal to the VM
runbook.
