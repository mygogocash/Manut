## P0 — HIGH severity, verified REAL

### H1. Projects Kanban view crashes (`ReferenceError`) — `bug`, Effort S

- **Surface:** Projects page → "Kanban view" toggle.
- **File:** `packages/frontend/core/src/desktop/pages/workspace/projects/index.tsx:526,643,649`
- **Mechanism:** `readableStatus()`, `priorityClass()`, `formatDueDate()` are
  called in `index.tsx` but defined **only** in sibling `detail.tsx:65,89,105`
  and never imported. A refactor moved them ("ProjectTasksSection has moved
  into ./detail.tsx") and dropped the import — the §2.2/§6 dropped-dependency
  scar.
- **Verified on 3 axes:** source (no import), `tsc --noEmit` → `TS2304: Cannot
find name 'readableStatus'` ×3, and the shipped `dist/.../projects/index.js`
  carries the broken refs with no local def. Bundle was produced by a
  transpile-only path (swc/isolatedModules) so it sails past bundling and only
  throws at runtime when the user toggles Kanban.
- **Fix:** `export` the 3 helpers from `detail.tsx` and import them in
  `index.tsx`, or extract to `projects/helpers.ts` imported by both. Add a
  render smoke-test for the Kanban toggle.

### H2. Social connections write to the wrong table — metrics never populate — `bug`, Effort L

- **Surface:** Settings → Connections → connect Facebook/IG/Threads/TikTok/LINE.
- **Files:** `desktop/dialogs/setting/workspace-setting/analytics-connections/panel.tsx:889-974`
  (writes `IntegrationConnection`) vs `plugins/analytics/.../meta.poller.ts:67`
  (reads `SocialConnection`).
- **Mechanism:** Two **disjoint** social-connection systems. The mounted
  settings panel persists to `IntegrationConnection`; the pollers that fill
  `social_metrics` read `SocialConnection`, populated only by the _other_
  system (`plugins/analytics/connections/connection.service.ts`). Nothing
  bridges them → user connects Facebook, sees "Connected", overview stays
  empty forever.
- **Verified REAL** end-to-end (resolver → service → table, both sides).
- **Fix (needs a decision):** pick ONE social-connection system and delete the
  other, or have the `*-oauth` callbacks also upsert a `SocialConnection` row.

### H3. 4 connection resolvers skip workspace authz — `backend/security`, Effort S

- **Surface:** PostHog / GoGoCash / Facebook / generic Connections mutations.
- **Files:** `posthog-connection.resolver.ts:83-159`, `gogocash-connection.resolver.ts:62-119`,
  `facebook-oauth.resolver.ts:65-121`, `connections/connections.resolver.ts:37-58`.
- **Mechanism:** guard only `if (!user)`, then pass caller-supplied
  `workspaceId` straight through — no `AccessController.assert(...)`. Siblings
  (`mongodb-connection.resolver.ts:100-103`, `analytics/connections/connection.resolver.ts:56-59`)
  do gate on `Workspace.Settings.Update`.
- **Bounded** to same-user data pollution (rows keyed `userId_workspaceId_provider`),
  **not** cross-tenant token theft — but a clear deviation from the team's own
  pattern. **Verified REAL.**
- **Fix:** add `ac.user(id).workspace(wsId).assert('Workspace.Settings.Update')`
  to the `set*`/`disconnect*` mutations and `'Workspace.Read'` to the queries
  in all four.

### H4. Knowledge Graph: uncapped O(n²) physics + full repaint every frame, never pauses when hidden — `perf`, Effort M

- **File:** `modules/knowledge-graph/views/graph-view.tsx:576-982` (loop at `:587-603`, reschedule `:971`).
- **Mechanism:** all-pairs repulsion over every non-trash doc, no node cap, no
  settle/cooldown, no `document.hidden` gate (sibling `graph-mini.tsx:116` does
  gate). Twinkle uses `performance.now()` so it repaints at 60fps forever even
  after layout settles. Heaviest CPU/GPU/battery surface in the app.
- **Verified REAL.**
- **Fix:** (1) pause on `document.hidden`; (2) cap to top-N by degree or
  Barnes-Hut/spatial-grid for large graphs; (3) kinetic-energy threshold to
  stop scheduling `step()` at rest, resume on interaction/pulse.

### H5. No `prefers-reduced-motion` anywhere in Manut UI — `a11y`, Effort M

- **Files:** `graph-view.tsx` + `graph-mini.tsx` (canvas loops); `analytics/views/ai-strategist/index.css.ts:4`,
  `analytics/views/platform-page/index.css.ts:4` (`skeletonShimmer`),
  `analytics/components/insight-card/index.css.ts:4` (`fadeIn`).
- **Mechanism:** grep for `prefers-reduced-motion|useReducedMotion` across all
  Manut module dirs → **zero**. WCAG 2.3.3 / 2.2.2 gap for vestibular users.
- **Verified REAL.**
- **Fix:** `@media (prefers-reduced-motion: reduce)` → `animation: none` on the
  keyframe blocks; `matchMedia('(prefers-reduced-motion: reduce)')` guard to
  render a single static frame in the canvas loops.

### H6. "View run" link navigates behind the still-open Settings modal — `bug`, Effort S

- **Surface:** Settings → Control Plane Roles → Agent roles → "view run".
- **File:** `workspace-setting/integration/control-plane-roles/setting-panel.tsx:217-222`.
- **Mechanism:** `workbench.open('/release-runs/${id}')` defaults to
  `at:'active'` (pushes under the modal); the Settings `Modal` is never closed,
  and `ControlPlaneRolesSettingPanel` is mounted without an `onCloseSetting`
  prop so it _can't_ close the dialog. User clicks → nothing visibly happens.
  Sibling Gmail import correctly uses `{at:'new-tab'}` + toast.
- **Verified REAL.**
- **Fix:** thread `onCloseSetting` down and call it before `workbench.open`, or
  open in a new tab + confirmation toast.
